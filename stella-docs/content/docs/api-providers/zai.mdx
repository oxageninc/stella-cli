<ProviderMark id="zai" size={30} />

Z.ai serves the GLM family, and `glm-5.2` is Stella's **default worker tier**: excellent
coding at a fraction of flagship price ($0.60/$2.20 per Mtok against $3.00/$15.00). Z.ai
sits **first** in [auto-detection](/docs/api-providers#auto-detection) — if a `ZAI_API_KEY`
is present, it is the provider you get without asking. Stella speaks the
**OpenAI-compatible** dialect against `https://api.z.ai/api/paas/v4`.

## Setup

```bash
export ZAI_API_KEY="..."
```

Alternatively, store the key in
[`~/.config/stella/credentials.toml`](/docs/configuration/credentials), or reference an
env var from [`settings.json`](/docs/configuration/settings):

```json title=".stella/settings.json"
{
  "providers": {
    "zai": {
      "api_key_env": "MY_TEAM_ZAI_KEY"
    }
  }
}
```

## Selecting models

The provider id is `zai`; the default model is `glm-5.2`.

```bash
stella --model zai/glm-5.2 run "implement the pagination endpoint"
export STELLA_MODEL=zai/glm-5.2
```

## The coding-plan endpoint

If you subscribe to a Z.ai **GLM coding plan**, your quota lives on a dedicated endpoint —
the flat subscription that makes `glm-5.2` a workhorse you stop metering. Route Stella to
it either way:

```bash
export ZAI_GLM_CODING_PLAN=1
```

or, provider-agnostically, in [`settings.json`](/docs/configuration/settings):

```json title=".stella/settings.json"
{
  "providers": {
    "zai": {
      "base_url": "https://api.z.ai/api/coding/paas/v4"
    }
  }
}
```

<Callout type="warn">
  If both are set, the `ZAI_GLM_CODING_PLAN=1` env toggle wins over the `settings.json`
  `base_url` — unset the toggle when you switch to the settings route, or the file value
  is silently overridden. Details on the precedence chain live in
  [settings.json](/docs/configuration/settings).
</Callout>

## Thinking

GLM thinking maps directly from Stella's reasoning setting: `reasoning: "on"` sends
`thinking: {"type": "enabled"}` on the wire, `"off"` sends `{"type": "disabled"}`. Set it
per agent in [`agent_engine_config`](/docs/configuration/agent-engine-config).

## In the pipeline

`glm-5.2` is the worker of the [balanced profile](/docs/examples/balanced) —
flagship-adjacent coding under a `claude-fable-5` or `gpt-5.5` judge — and the judge of the
[dirt-cheap profile](/docs/examples/dirt-cheap) over a `deepseek-chat` worker. On a coding
plan, it is the obvious default for high-volume fleet work. Per-agent wiring lives in
[`agent_engine_config`](/docs/configuration/agent-engine-config).
