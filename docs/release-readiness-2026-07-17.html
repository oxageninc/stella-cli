03 Fixed this pass

Every fix ships with a witness test (fails on old code, passes on new) per Stella's own definition of done.

P0 · security

Untrusted repo can't redirect your key

A cloned repo's project-scope settings.json could silently override a built-in provider's base_url or repoint api_key_env — exfiltrating your real key to an attacker host on the first call. Now gated behind STELLA_TRUST_PROJECT, like hooks.

P0 · security

No writes through a dangling symlink

resolve_within_root walked ancestors with exists(), which follows links — a dangling symlink read as a new in-root file and let write_file escape the workspace. Now walks with symlink_metadata and rejects outward links.

P1 · memory

Self-improvement recall, un-broken

A/B recall suppression keyed off wall-clock nanoseconds — on macOS's microsecond clock it fired on 100% of turns, disabling recall entirely. Replaced with a real turn counter.

P1 · memory

Reflection fires on the run path

The pipeline kept tool turns out of the shared transcript, so the reflection gate was always false — the loop never learned on stella run. Now gated on real work + fed the final answer. Proven end-to-end.

P1 · reliability

Sessions don't brick on empty text

The Anthropic adapter could serialize empty/whitespace text blocks (a 400 that recurs every replay). Now emits text only when non-blank; drops content-less turns.

P1 · tools

grep finds -> and reports errors

Patterns starting with - were parsed as flags, and rg's exit-2 was swallowed as "no matches." Now passed with -e, with real pattern errors surfaced and partial results kept.

05 Where Stella stands out — and where peers lead

Stella's edge

  • Verification as a contract. Witness tests and verify_done's shadow-worktree replay make "done" falsifiable — most agents accept a green suite; Stella won't.
  • Privacy by construction. No phone-home is an architectural invariant, not a setting. All telemetry is local SQLite — and now has a dashboard.
  • Truly vendor-neutral. Nine providers plus any local OpenAI-compatible endpoint, one credential chain, no account.
  • A reason-about-able engine. stella-core does no I/O, so the loop is property-testable — rare in this class.

Where the field is ahead

  • Aggregate cost governance. Fleet honors a per-child cap but not a true aggregate budget — a wide fan-out can overspend the flag. (Open P1.)
  • Test-gap surface. ~20 critical behaviors (turn driver, SSE parsing, settings merge, TUI wrap math) still lack direct tests.
  • Polish under load. A few TUI key-routing and metering-display bugs betray a young interactive layer.
  • Docs freshness. ~115 comments cited removed internal spec files (being swept this pass); the registry-of-truth discipline is still settling.
