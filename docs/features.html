Stop trusting your coding agent. Watch it work.

Stella is an open-source coding agent that runs in your terminal: one Rust binary, no SaaS, your own API keys. Its Command Deck puts the agents, the live traces, the code graph, the skills, and the MCP servers on one screen. Every model call priced, every tool call timed, and all of it stored on your disk.

▸ AGENTS

Every agent, every toolbelt, in plain sight.

Subagents usually run on faith: you find out what tools they had after something breaks. The Agents tab lists all 15 installed agents with their scope, their pinned version, and the exact toolbelt each one is allowed to use. A reviewer gets Read, Grep, Glob, and Bash. A break-fix agent gets Edit and Write too. Editing an agent creates a new pinned version, so a prompt never changes under you.

▸ TRACES

Every model call priced. Every tool call timed.

An AI bill should not be a monthly surprise. Traces streams the run as it happens: 96 events in this session, each model call with its own cost, down to $0.0065, and each tool result with its wall time, down to the millisecond. The running total and the cache hit rate, 81% here, sit at the bottom of the screen the whole time. It all lands in a local SQLite file you can open with any client.

▸ GRAPH

It queries a code graph instead of grepping.

Grep tells you where a string appears. The graph tells you what a file defines and what depends on it. Stella parses your repo with tree-sitter into an offline code graph: no embeddings, no API key, and no network. This one file resolves to 176 nodes and 175 relations, with an ASCII map of the neighborhood rendered right in the terminal.

▸ SKILLS

Skills install like packages, not pasted prompts.

A folder of prompts does not survive a second machine. Stella treats skills as scoped, versioned packages: 17 installed here across user and project scope, toggled with a keypress. Search the registry without leaving the deck, preview a skill before it lands, and install it in place. The top result carries 482.6K installs, so you can see what the rest of the ecosystem actually uses.

▸ MCP

MCP servers, found and connected from the deck.

Adding an MCP server usually means hand-editing JSON and restarting. The MCP tab searches the registry live: remote, npm, and pypi servers alike, each with a plain description of what it does. One search for Stripe returns ten servers, from the official remote to billing, analytics, and checkout bridges. Pick one, press enter, and it joins the tool registry in place.

▸ ALWAYS ON

The meter never leaves the screen.

Every tab shares one bottom line: agent, stage, model, CPU, context, spend, cache hit rate, and active agents. Above it, the run moves through plan, execute, and verify, and verify is a stage, not a promise: a change must fail on the old code and pass on the new before Stella calls it done. The prompt queue never blocks, and ! drops you into a shell. Stella calls this deterministic-first, and prints it in the corner to hold itself to it.
